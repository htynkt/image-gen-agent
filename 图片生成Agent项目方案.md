# 图片生成 Agent 项目方案

> 版本：v1.0　|　日期：2026-06-24　|　作者：茜茜 & Claude
>
> 一句话定位：**一个能陪你聊天、记住你偏好、并按你的指令生成三种图（拼豆图 / AIGC 创意图 / 人像转卡通娃娃）的个性化图像 Agent。**

---

## 一、项目概述

### 1.1 它解决什么问题
现在生成一张图通常要：开工具 → 想提示词 → 调参 → 等结果 → 不满意重来。本 Agent 把这个过程变成**自然对话**：
- 你发一张照片或一句话（"把这张自拍变成盲盒娃娃"、"拼豆版的小柴犬"、"赛博朋克风的猫"）；
- 它理解意图、自动选对应的生图能力、套用**你自己的风格偏好**，直接出图；
- 它**记住**你的喜好和过往作品，下次越用越顺手。

### 1.2 三大核心能力
| 能力 | 输入 | 实现路径 | 成本 |
|------|------|----------|------|
| **拼豆图** | 图片 / 关键词 | 本地算法（像素化 + 拼豆色板最近邻匹配） | 免费 |
| **AIGC 创意图** | 关键词 / 描述 | 国产文生图 API | 按次计费 |
| **人像转卡通娃娃** | 人像照片 | 国产图生图 API + 提示词模板 | 按次计费 |

### 1.3 设计原则
1. **原型先行**：先在本地用 Gradio 跑通完整闭环，验证效果，再考虑产品化。
2. **本地优先，API 兜底**：能本地算的（拼豆）绝不花 API 钱；需要大模型的（AIGC、人像）走国产合规 API。
3. **个性化是一等公民**：语气人设、默认风格、长期记忆从第一天就设计进去，不是后加的。
4. **模块化**：三个生图能力是独立"技能（Skill）"，对话引擎是调度者，互不耦合。

## 四、核心模块详细设计

### 4.1 对话引擎（大脑）
- **职责**：理解你要什么、决定调哪个技能、用你的语气说话。
- **实现**：基于 LLM 的 **工具调用（Function Calling）**。把三个生图能力注册成"工具"，LLM 自己判断该调哪个、传什么参数。
- **关键**：每轮对话前，把「人设 + 默认风格 + 检索到的相关记忆」拼进系统提示词。

```python
# 伪代码：系统提示词组装
system_prompt = f"""
你是{profile.name}的专属图像助手，称呼 ta 为「{profile.nickname}」。
语气风格：{profile.tone}          # 如：温柔、活泼、偶尔撒娇
默认画风偏好：{profile.style}     # 如：马卡龙配色、圆润线条

你可以使用以下工具：
- generate_bead_art(image_or_keyword, size)  拼豆图
- generate_aigc_creative(keyword, style)     AIGC 创意图
- portrait_to_cartoon(image, style)          人像转卡通

【相关记忆】
{retrieved_memories}
"""
```

### 4.2 意图路由（避免误判）
用户一句话可能有歧义（"来个柴犬" 可能要拼豆也可能要创意图）。策略：
1. **优先看输入类型**：带照片 → 大概率是"转拼豆"或"转卡通"；纯文字 → 大概率"AIGC 创意图"或"拼豆"。
2. **关键词规则兜底**：含"拼豆/图纸/材料"→ 拼豆；含"娃娃/盲盒/卡通我"→ 人像转卡通。
3. **不确定就问一句**：LLM 判断置信度低时，反问"你是想要拼豆版还是创意图呀？"——比硬猜出错好。

### 4.3 拼豆图技能（本地算法，重点）
**这是唯一不花 API 钱、完全可控的能力，建议第一个做。**

输入图片 → 输出三件套：① 像素化拼豆效果图 ② 拼装对照图（带网格+色号）③ 材料清单（每色多少颗）。

**算法步骤**：
1. **载入标准拼豆色板**：内置一套常用色板（如 Artkal C 系列 / Perler 29 色），存成 `[{name, code, rgb}]` 的 JSON。
2. **缩放到拼豆板尺寸**：按目标格子数（如 29×29、50×50）等比缩放原图。
3. **颜色量化匹配**：对每个像素，在色板里找**色差最小**的颜色。⚠️ 用 **CIEDE2000 色差公式**（在 Lab 空间算）比 RGB 欧氏距离准很多，皮肤和过渡色不会跑偏。
4. **渲染拼豆图**：用圆形画豆子（带一点高光），接近真实拼豆质感。
5. **生成对照图 + 统计材料**：网格上标色号，统计每种颜色用量。

```python
# 核心匹配逻辑伪代码
def match_bead(pixel_rgb, palette):
    pixel_lab = rgb2lab(pixel_rgb)
    # 找色板中 CIEDE2000 色差最小的
    return min(palette, key=lambda b: ciede2000(pixel_lab, rgb2lab(b.rgb)))
```
> 关键词输入时：先用文生图 API 生成一张基础图，再走上面流程转拼豆。

### 4.4 AIGC 创意图技能（文生图 API）
- **流程**：用户关键词 → 引擎做**提示词增强**（补充细节、套用你的默认风格）→ 调文生图 API → 出图。
- **提示词增强示例**：
  - 你说："赛博朋克的猫"
  - 增强后："赛博朋克风格的一只猫，霓虹蓝紫色调，雨天街道反光，{你的默认风格：马卡龙配色}，高清，电影感"
- **推荐 API**：智谱 CogView-4 / 字节即梦（豆包 Seedream）/ 可 KLING——国内合规、对中文提示词理解好。

### 4.5 人像转卡通娃娃技能（图生图 API，难度最高）
- **流程**：人像照片 → （可选）人脸检测+抠图/居中 → 图生图风格化 → 后处理。
- **提示词模板**（泡泡玛特盲盒风）：
  > "泡泡玛特盲盒公仔风格，3D 渲染，Q 版大头小身，{你的特征：保留发型/眼镜}，柔和塑料质感，{你的默认配色}，影棚打光，纯色背景"
- **稳定性技巧**：固定一套参考提示词模板，只换"人物特征"变量；保留人脸特征靠**参考图/ControlNet**（如 API 支持）；出多张让用户挑。
- ⚠️ 人脸照片属敏感数据，**只传 API、不落库**，本地仅缓存生成结果。

### 4.6 个性化记忆系统（让 Agent 越用越懂你）

**两层结构**：

**① 静态档案 `config/profile.yaml`**（你手动维护，一次性配置）
```yaml
nickname: 茜茜          # 怎么称呼你
tone: 温柔、带点小俏皮，偶尔用颜文字  # 对话语气
style:                   # 默认风格偏好
  palette: 马卡龙色（粉、薄荷绿、奶油黄）
  vibe: 圆润、治愈、明亮
  avoid: 高饱和荧光色、恐怖元素
bead:                    # 拼豆默认参数
  default_size: 29
  palette: artkal_c_90
```

**② 动态长期记忆 `data/memories.jsonl`**（Agent 自动写）
- 每轮对话结束后，用一个**轻量 LLM 调用**抽取"值得记住的事实"（如："茜茜喜欢柴犬"、"上次赛博猫用的是蓝紫调"、"讨厌荧光粉"）。
- 存成结构化条目：`{id, content, tags, related_outputs, time}`。
- 下一轮对话前，按当前话题**检索 Top-K 相关记忆**注入提示词（简单版用关键词匹配，进阶版用向量检索）。
- 这套机制直接借鉴 Claude Code 自身的 memory 设计，已验证可行。

## 五、技术栈选型

| 层 | 选型 | 理由 |
|----|------|------|
| 原型界面 | **Gradio** | 几行代码搞定"聊天框+图片上传+图片展示"，Python 原生，改 Web 时可复用 |
| 对话 LLM | **智谱 GLM-4.5 / GLM-4-Plus** | 国内合规、中文好、支持 Function Calling、有免费额度 |
| 视觉理解（看用户发的图） | **GLM-4V** | 判断照片内容、辅助人像检测 |
| 文生图 / 图生图 | **智谱 CogView-4 / 即梦 / 可灵** | 国产合规，中文提示词友好 |
| 图像处理（拼豆算法） | **OpenCV + Pillow + scikit-image** | 像素化、色板匹配、Lab 色差、渲染豆子 |
| 记忆检索（进阶） | **Chroma / FAISS** | 向量检索相关记忆；MVP 阶段可先用 JSON+关键词匹配 |
| 配置 | **PyYAML** | 个性化档案用 yaml，人好读好改 |
| 语言 | **Python 3.10+** | 生态最全 |

## 八、关键难点与应对

| 难点 | 风险 | 应对 |
|------|------|------|
| 拼豆颜色失真 | 肤色/渐变跑偏 | 用 CIEDE2000 而非 RGB 距离；必要时加抖动(dithering) |
| 意图误判 | 该出拼豆却出了创意图 | 输入类型 + 关键词规则 + 不确定就反问 |
| 人像转卡通不稳定 | 每次风格不一致 | 固定提示词模板、出多张候选、支持参考图 |
| API 成本失控 | 反复试图烧钱 | 加每日调用上限、失败重试限次、本地缓存相同请求 |
| 人脸隐私 | 照片泄露 | 照片不落库、仅传 API、生成结果本地存 |
| 记忆噪声 | 记了一堆没用的 | 抽取记忆时设严格标准 + 支持手动清理 |

## 九、成本与合规

- **API 成本**：GLM 有免费额度足够原型；文生图/图生图按次计费（约几分级一张），原型阶段月成本可控在几十元内。
- **合规**：全程国产 API，数据不出境，符合国内监管。
- **隐私**：人像照片明文不存储，仅做生成用途。
